Ruby Arrays vs. Hashes

July 13, 2015

Implementation Details

The Ruby docs explain that an Array is "an ordered,integer-indexed collection of any object."

The key point here when distinguishing an Array from a Hash is the term "integer-indexed". Each element of an Array is given a specific index, which will be an integer ranging from 0 to the length of the array minus 1.

For example, declare an array:
my_array = ["Hurrah", 2.0, 10]

We can then access each item of the array using its integer index: puts my_array[0] #prints "Hurrah"

Hashes on the other hand are described by Ruby docs as a "dictionary-like collection of unique keys and their values. Also called associative arrays, they are similar to Arrays, but where an Array uses integers as its index, a Hash allows you to use any object type."

The notable difference here is that the keys of a Hash cah be any object type, whereas an array must be indexed with integers. For example, see how we declare and access the individual items of a hash:

options = { font_size: 10, font_family: "Arial" } options[:font_size] # => 10

Usage

Arrays and Hashes are used for different tasks.

An Array is useful for multi-valued properties. For example, suppose the property is my favorite music. This can included multiple values, i.e.

music1 = "blues"
music2 = "rock"
music2 = "bollywood"

Now rather than having 3 different variable to store all of my music tastes, we can use an array: music = ["blues", "rock", "bollywood"]

Hashes on the other hand, are useful for creating data structures. For example, suppose I want to represent all the properties of a musical band. I could create a single Hash with properties for singer, drummer, albums (which could be an array):

band = { singer: "Jared", drummer: "Goutham" albums: ["LA heartbreaks", "Cloud songs", "Plant Blues"] }
